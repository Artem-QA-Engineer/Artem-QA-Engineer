# Привет!
Я начинающий специалист по тестированию.  
В этом репозитории собраны примеры моей тестовой документации, демонстрирующие навыки написания тестовой документации.

## Навыки и инструменты
- Тест-дизайн: Классы эквивалентности, граничные значения
- Документация: Чек-листы, тест-кейсы, баг-репорты, тест-план
- Инструменты: GitLab, GitHub, Postman, Charles Proxy, DevTools, Android Studio, VS Code, InteliJ IDEA, PyCharm, pgAdmin, Jira, Яндекс Трекер
- Языки программирования: основы SQL, Java, Python, C, Bash
- Дополнительно: Понимание клиент-серверной архитектуры, REST API 

## Что внутри:
- **[Чек-листы](./checklists/)** — пример проверок для веб-форм и мобильных приложений.
- **[Тест-кейсы](./test-cases/)** — детальные сценарии для страницы логина.
- **[Баг-репорты](./bug-reports/)** — примеры оформления найденных ошибок.
- **[Тест-план](./test-plan/)** — документ, определяющий объект, стратегию, ресурсы, сроки и критерии проверки.
